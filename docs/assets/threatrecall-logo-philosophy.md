# Threat Recall Logo Philosophy

## Signal Memory

Threat Recall needs a mark that feels analytical without becoming decorative: a compact system where detection, memory, and cyber threat context are all present in a few precise lines. The visual language should be restrained, technical, and quiet. It should feel like a tool built for analysts who trust clarity over noise.

The core form is a shield-derived hexagon, preserving the security cue already present in the repository's existing ZettelForge icon while making it less generic through a recall loop inside the perimeter. The loop suggests retrieval, supersession, and memory evolution; the three small nodes suggest CTI entities and graph relationships. Nothing is literal enough to become a mascot or diagram.

Color is limited to graphite, white, and the existing neon green accent. The green should act as signal, not decoration: one controlled highlight that implies a live indicator or confirmed finding. The mark must work in one color, on dark backgrounds, and at favicon scale.

The wordmark should be simple and uppercase, with wide tracking and a clean technical sans serif. It should not overpower the icon. The result should feel like a product company rather than an open source package badge: sparse, legible, and memorable.
