## Lessons Learned — 2026-04-02 17:15 CDT

### Hardcoded Paths Break After Relocation
The entire darkweb-observatory suite had `/var/www/html/` hardcoded across 10+ scripts. This pattern causes silent failures any time the suite runs outside the original deployment environment. The fix was `sed -i 's|/var/www/html/|./output/html/|g'` across all `.py` files.

Pattern: always use relative paths or env var substitutions for file paths. Never hardcode absolute paths in scripts that move between machines or users.

### Model Pull ≠ Model Change
`ollama pull` can re-pull the same model tag and produce different vector outputs if the underlying model implementation changed. The `nomic-embed-text-v2-moe:latest` was re-pulled at 14:38 today, updating its digest. Any production vector database built with the old digest is now stale. Mitigation options: pin model digests in config, maintain a model version file, or accept re-indexing after model updates.

### Model Update Detection
`ollama list` shows `MODIFIED` timestamp. After a pull, `MODIFIED` updates even if the tag is the same. Best practice: after any `ollama pull`, check `MODIFIED` timestamp and compare against your last known-good timestamp. If changed and you have a populated vector DB, trigger re-index.

### LanceDB Table Existed But Was Empty
The `vector_memory.lance` database had a `memories` table created but never populated. The LanceDB integration was built as a concept but never wired into the live pipeline. This means: (1) no stale index to rebuild, and (2) the vector store was never actually in production use. The active recall system relies on `notes.jsonl` + `embedding_utils.py` on-demand generation, not a pre-built index.

### Security: Per-Agent Exec Approvals
Exec approvals for agents are controlled at TWO levels:
1. `tools.exec.security` (global) — set to `full` to bypass all exec approvals
2. `channels.<channel>.accounts.<account>.execApprovals.enabled` (per-account, Telegram) — controls whether in-chat `/approve` buttons appear, NOT whether approval is required

Removing `claude-cli` fallback models (4x `model_not_found` → `FailoverError` on every turn) from `agents.defaults.model.fallbacks[]` fixed the IPC bridge saturation between Patton and Tamara. These fallbacks caused all lane task turns to hit 4 consecutive errors before reaching valid models.
