# ThreatRecall API Changes - 2026-04-03

## Summary
Production-ready implementation connecting the FastAPI wrapper to the actual A-MEM memory system with tenant isolation, OCSF audit logging, and proper error handling.

## Files Changed

### 1. `src/threatrecall_api/api/routes/memory.py`
**Status:** MOCKS REPLACED WITH REAL IMPLEMENTATION

#### Changes:
- **Health endpoint** (`/{tenant_id}/health`):
  - Added tenant existence check (404 if not found)
  - Real stats: note count from MemoryManager, storage bytes from filesystem
  - Status: "healthy" vs "degraded" based on actual storage state
  
- **Remember endpoint** (`/{tenant_id}/remember`):
  - Connected to real `MemoryManager.remember()` pipeline
  - Source type mapping: "cisa" → authoritative, "operator" → conversation
  - Returns full note with entities and linked notes
  - 409 CONFLICT for duplicates
  - Proper error codes per GOV-005
  
- **Recall endpoint** (`/{tenant_id}/recall`):
  - Connected to real `MemoryManager.recall()` with vector search
  - Returns scored results with entity extraction
  - Cursor pagination support
  - Includes links when requested

- **Compliance:**
  - GOV-005: Standard envelope, error codes, cursor pagination
  - GOV-012: OCSF audit logging
  - Proper HTTP status codes

### 2. `src/threatrecall_api/main.py`
**Status:** AUDIT + RATE LIMITING MIDDLEWARE

#### Changes:
- **RateLimitMiddleware**: Added for all requests
- **AuditMiddleware**: Replaced inline middleware with OCSF class 6002 logging
  - Extracts tenant_id from path for audit context
  - Logs request_id, tenant_id, method, path, status, latency, IP
  - Uses `log_api_activity()` helper
  - Per GOV-012 (OCSF audit logging)
  
- **Validation handler**: Improved field extraction for error messages
  
- **Root health check** (`/health`): Added for load balancers

- **Removed**: Inline middleware replaced with proper middleware classes

### 3. `src/threatrecall_api/models/note.py`
**Status:** TLP CLASSIFICATION FIX

#### Changes:
- Fixed enum name: `TLPC classification` → `TLPC` (syntax error)
- Fixed TLP format strings:
  - `TLP_WHITE` → `"TLP:WHITE"` (was `"TLP_WHITE"`)
  - `TLP_GREEN` → `"TLP:GREEN"`
  - `TLP_AMBER` → `"TLP:AMBER"`
  - `TLP_AMBER_RED` → `"TLP:AMBER+RED"`
  - `TLP_RED` → `"TLP:RED"`
